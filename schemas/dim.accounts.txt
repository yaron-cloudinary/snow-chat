Below is a list that describes the dim.accounts database table.  Each row describes a column.   The first word is the database column name.  The rest of the row is a comment.  Not all have comments.  The comment describes the column.


ID	
NAME	The official name of the account, generated from the admin > sfdc > netsuite > top cloud

MONTHLY_ACTIVE	MAU Active indication - at least 1 DAU in 30d; #active #activation
WEEKLY_ACTIVE	WAU Active indication - at least 1 DAU in 7d; #active #activation
DAILY_ACTIVE	DAU Active indication (usage) - intent API calls, transformations, image impressions > 2.5K, or video seconds > 3H; #active #activation

WEEKLY_ACTIVE_COMPONENTS	WAU Broken by Transformations, API, Console, Media Library, and Bandwidth; #active #activation
DAILY_ACTIVE_COMPONENTS	DAU Broken by Transformations, API, Console, Media Library, and Bandwidth; #active #activation

PAYING	Paying indication - represents MRR existence, removing internal and poc accounts; #mrr
MRR_DOLLAR	Main revenue value for the account, calculated by plan price+overages and addon price+payg addon price, including discounts and annual forecast; #mrr

CS_SEGMENT_LEVEL	CS enterprise-only account segmentation, usually for Scaled; #segment
PLAN_TYPE	Annual Bucket vs Monthly Units / Legacy
UNIT_TYPE	Credits (1K) vs Units (1M) Unit Size
SW_DOMAIN	Primary domain of the account, sorted by value origin, monthly visits, and industry value; #domain
